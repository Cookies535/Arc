# Domain Glossary

- **Node (节点)**: 3D 星图中的核心实体，代表一个记忆或事件。包含日期、标题、描述、类型以及 3D 坐标（X=时间，Y=高度/层级，Z=深度）。
- **Photo (照片)**: 导入的本地图片。与 Node 之间是多对多（Many-to-Many）关系，仅被引用。
- **Thumbnail (缩略图)**: 导入时生成并拷贝到应用内部沙盒（`thumbnails/` 目录）的高质量小图（如 400x400），按文件 Hash 命名。用于 3D 星图和画廊的高性能展示，防止原图移动导致死链。
- **Original URI (原图引用)**: 数据库中记录的照片原图绝对路径。仅在用户“查看原图”或重新定位时使用。若失效，界面通过半透明遮罩提示“原图不可访问”。
- **Archive Package (.arc 数据包)**: 跨端同步的实体，包含 SQLite 数据库、所有的缩略图和节点数据。导入时保持星图结构完整，但允许 Original URI 处于失效状态（待用户批量重新定位）。
- **Starfield View (星图视图)**: 使用 Flame 引擎实现的 2.5D 核心视口，负责渲染浮动平台、发光轨道、粒子系统和背景视差。
- **UI Overlay (覆盖层 UI)**: 纯 Flutter Widget 构成的常规界面（如详情页、设置、画廊、液态玻璃弹窗），叠加在 Starfield View 之上。
- **Cluster (星团)**: 当节点数量过载或镜头拉远时，相邻节点聚合显示为星团状态，点击后展开。
- **Chunk (时间区块)**: 内存和渲染管理单位。星图视口只加载镜头附近特定时间跨度（如 ±3 个月）的节点。
- **Shadow Data (影子数据)**: 预留给未来 AI 引擎的独立数据字段（如 `ai_suggested_tags`, `ai_description`）。在用户明确点击“采纳”前，绝不覆盖用户的手动输入，在 UI 上以“✦”标识。
- **Processing Queue (处理队列)**: 基于 Dart Isolate 实现的后台队列，负责在不阻塞主 UI 的情况下，对缩略图执行 K-Means 主色调提取和 EXIF 解析。
- **Track / Connection (轨道/连接)**: Node 之间的连线。分为默认的“时间顺序自动串联轨道”（人生主轨道）和手动添加的“主题连接”（如通过颜色或虚线区分的逻辑关联）。
- **Node Type (节点类型)**: 节点的分类（如日常、旅行、里程碑、计划），它决定了节点在 3D 星图中的平台外观和光晕颜色。
- **Unknown Starfield (未知星域)**: 针对缺失 GPS 数据的节点设置的默认存放区域，用户可在此区域或 2D 地图上进行手动定位。
